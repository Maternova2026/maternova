# maternova
1. Introduction
The Goal-Based Software Measurement (GQM) Framework is a structured approach used to define and interpret software metrics based on specific goals. It ensures that measurement activities are aligned with the objectives of the system being developed.
For the Maternova Web Application, the GQM framework is used to evaluate the system’s usability, performance, reliability, and effectiveness in capturing maternal healthcare data.

The framework follows three hierarchical levels:

1. Goal – What we want to achieve.

2. Question – Questions that determine whether the goal is achieved.

3. Metric – Quantitative measurements used to answer the questions.

2. System Overview

Maternova is a simple web-based application designed to capture and manage information about pregnant mothers. The system allows healthcare personnel to:

Register pregnant mothers

Record maternal health data

Track appointments

Store contact and emergency information

Monitor pregnancy-related health indicators

The system aims to improve maternal data collection, patient tracking, and healthcare decision making.

3. GQM Framework Implementation
Goal 1: Improve Data Capture Efficiency

Goal:
Ensure that maternal data can be captured quickly and accurately by healthcare staff.

Questions

How long does it take to register a new patient?

How many data entry errors occur during registration?

Is the data entry interface easy for healthcare workers to use?

Metrics

Average Patient Registration Time

Total registration time / Number of registrations

Data Entry Error Rate

Number of incorrect records / Total records submitted

Form Completion Rate

Number of completed forms / Number of started forms
Goal 2: Improve System Usability

Goal:
Ensure the application is easy to learn and use by healthcare workers.

Questions

Can users navigate the system easily?

How many clicks are required to complete key tasks?

Do users experience difficulties during use?

Metrics

Average Task Completion Time

Number of Clicks to Register Patient

User Satisfaction Score (collected through surveys)

Example:
| Metric                | Measurement Method                  |
| --------------------- | ----------------------------------- |
| Navigation Efficiency | Average number of clicks per task   |
| Task Completion Time  | Time taken to complete registration |
| User Satisfaction     | Survey rating from 1–5              |
